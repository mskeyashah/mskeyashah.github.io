At HackDavis 2020, my team and I built a mobile application called VeggieBuddie. This app won 2nd place in "Most Social Good" and "Best Use of Google Cloud" categories and was one of the top 20 teams in Google Solution Challenge 2020 where over 800 universities participated worldwide.

The app is published with enhancements on Google Play Store as Veg N' Snap.

Development

Process

We made our app on Android Studio using Flutter and Dart programming. Using Google user authentication, the user can log into the app through their Google account to save any dietary preferences such as allergens. This data is stored in the Google cloud's realtime database, Firebase. We implemented Google's API for OCR to extract the text from the image of the ingredients. The text is then compared to the locally stored list of ingredients as well as the customized allergens list to determine a dietary restriction. You can also access the app offline by logging into the guest account.

Later on, we implemented a feature that whenever the app encounters an unknown ingredient, it uploads it to our database so that we can improve the accuracy. It also displays the ingredients broken down into vegan, vegetarian, and non-vegetarian as well as allergens if applicable.

This hackathon was sponsored by HCL and Microsoft companies to encourage development of applications that could help in the fight against corona virus. My team and I created a mobile app called COVID Trace that allows the users to monitor their health, know their risk, and report symptoms. We were in the top 32 teams that qualified for the semifinals in more than 200 teams that participated in the hackathon.

Development

Process

For developing the app we used flutter packages and dart programming language. First of all, we built the home screen with multiple resource links and an api that updates the global coronavirus count daily. Then we linked the self reporting page to it where users can report their test results or contact. We made sure to validate the answers given by the users. Once the answers were successfully recorded, we built the calendar page that retrieved the corresponding data from the answers and displayed a note each day for the next 14 days reminding the user to self isolate as well as report any symptoms they were feeling. Additionally, the app includes prevention tips as well reliable sources such as CDC and WHO. All the data is anonymous and stored on the user's phone to avoid any breach in privacy.

For MIT Hacks 2020, my team and I built a web-based Machine Learning project called Accessible Smart Linguist that detects American Sign Language using the live video feed from the camera and implements features like autocorrect and autocomplete to reduce the communication time between two people. Furthermore, the Learn feature of our application enables users to explore and improve their sign language skills in a fun and engaging way.

Development

Process

First of all, we self-created an image dataset for the 26 letters of the alphabet using OpenCV and python scripts. Then, we trained a Convolutional Neural Network using TensorFlow and Keras on the dataset for 50 epochs achieving about 98% accuracy. Next, we integrated the model into our website through the use of Python and Flask. The website takes in a live video stream from the camera and the user can sign in the designated space after which it is sent over to the server where our model accurately predicts the correct letter of the alphabet. Every time the user enters a space, the last word is autocorrected and if the word is incomplete, it is autocompleted. The user even has the option of manually clicking the autocomplete button to do the same. The user can start or pause anywhere as well as start with a clean palette. The Autocorrect/Autocomplete features are implemented through python libraries using an NLP model that trains as the user types. The front-end of the application was developed through the use of HTML and Flask templating.
